# GitHub Pages 部署指南

## 📋 前置准备

### 1. 安装 GitHub CLI（推荐）

```bash
# macOS 使用 Homebrew 安装
brew install gh

# 或者使用下载安装
# https://cli.github.com/
```

### 2. 配置 Git 用户信息

```bash
# 设置你的 GitHub 用户名和邮箱
git config --global user.name "你的GitHub用户名"
git config --global user.email "你的GitHub邮箱"
```

### 3. 认证 GitHub

```bash
# 使用 GitHub CLI 登录
gh auth login

# 或者配置 SSH Key
# https://docs.github.com/en/authentication/connecting-to-github-with-ssh
```

---

## 🚀 部署步骤

### 方法一：使用 GitHub CLI（最简单）

```bash
# 1. 进入项目目录
cd "/Users/zhangtao/WorkBuddy/排房系统"

# 2. 创建 GitHub 仓库并推送
gh repo create room-allocation-system --public --source=. --push

# 3. 启用 GitHub Pages
gh repo edit --enable-pages --branch main --path /

# 4. 等待 1-2 分钟，然后访问
# https://你的用户名.github.io/room-allocation-system/
```

---

### 方法二：手动在 GitHub 网站操作

#### 步骤 1：在 GitHub 创建新仓库

1. 访问 [https://github.com/new](https://github.com/new)
2. **Repository name**: `room-allocation-system`（或使用中文 `排房系统`）
3. 选择 **Public**
4. **不要**勾选 "Initialize this repository with a README"
5. 点击 **Create repository**

#### 步骤 2：推送代码到 GitHub

```bash
# 1. 进入项目目录
cd "/Users/zhangtao/WorkBuddy/排房系统"

# 2. 添加远程仓库（替换 你的用户名）
git remote add origin https://github.com/你的用户名/room-allocation-system.git

# 3. 推送代码
git branch -M main
git push -u origin main
```

#### 步骤 3：启用 GitHub Pages

1. 在 GitHub 仓库页面，点击 **Settings**
2. 在左侧菜单找到 **Pages**
3. **Source** 选择 **Deploy from a branch**
4. **Branch** 选择 **main** 和 **/ (root)**
5. 点击 **Save**
6. 等待 1-2 分钟
7. 访问：`https://你的用户名.github.io/room-allocation-system/`

## ❓ 常见问题

### 1. 推送时提示认证失败

**解决方法**：
- 使用 Personal Access Token (PAT)
- 生成 Token: https://github.com/settings/tokens
- 推送时使用 Token 作为密码

### 2. 页面显示 404

**解决方法**：
- 确认仓库是 **Public**
- 确认已启用 GitHub Pages
- 确认分支名称正确（main）
- 等待 5-10 分钟再试

### 3. 样式或功能不正常

**解决方法**：
- 检查浏览器控制台错误信息
- 确认所有资源路径正确
- 尝试硬刷新（Ctrl+F5 或 Cmd+Shift+R）
